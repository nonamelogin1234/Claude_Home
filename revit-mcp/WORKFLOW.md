# Правило работы с revit-mcp проектом

## Принцип

Работа ведётся **локально на ПК**. GitHub — только хранилище итога, не рабочий инструмент.

## Начало сессии

1. Клод читает CONTEXT.md из этой папки
2. Клод скачивает актуальные файлы проекта с ПК (или с GitHub если ПК недоступен)
3. Работаем, пишем код, тестируем — всё на ПК

## В процессе работы

- Клод пишет/правит файлы **напрямую на ПК** через shell API
- Сборка — `dotnet build -c "Debug R24"` на ПК
- Тестирование — в живом Revit
- В GitHub **ничего не пушим** до команды Сергея

## Конец сессии — только по команде Сергея

Когда Сергей говорит «пушь» / «сохрани в гит» / «заливай» — тогда и только тогда:
1. Клод читает финальные файлы с ПК
2. Пушит их в `Claude_Home/revit-mcp/src/`
3. Обновляет CONTEXT.md если нужно
