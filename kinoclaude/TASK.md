# TASK: Система кинорекомендаций «Посоветуй фильм»

## Цель

Реализовать в Claude Desktop сценарий:

1. Пользователь говорит «Посоветуй фильм»
2. Клод читает профиль киномана и задаёт 2–3 уточняющих вопроса (настроение, жанр, с кем смотреть и т.д.)
3. На основе ответов формирует критерии и идёт в БД — ищет подходящие фильмы среди тех, что пользователь **ещё не смотрел**
4. Подтягивает данные из Кинопоиска, веб-поиска и своей базы знаний
5. По очереди предлагает фильмы — не зачитывает синопсис, а **убеждает**: почему именно этот фильм подойдёт, опираясь на профиль вкусов

---

## Что уже есть

- MCP-сервер `kinoclaude_mcp.py` работает на VPS, подключён в Claude Desktop через `npx mcp-remote@latest`
- Инструменты: `search_films`, `get_film`, `get_film_staff`, `get_top_films`, `get_films_by_filters`
- PostgreSQL БД `jarvis_memory` доступна

---

## Что нужно сделать

### 1. Загрузить данные о просмотрах пользователя

В папке `kinoclaude/` лежит CSV-экспорт профиля с Кинопоиска (название, оценка, статус).

Нужно:
- Разобрать CSV и загрузить в таблицу `kinoclaude_ratings`
- Схему таблицы придумай сам исходя из задачи. Минимум: film_id (если есть), title, my_rating, status, genres, year
- Добавить поле `tags` — текстовый массив смысловых тегов (например: `["медленное кино", "экзистенциальный", "1990е"]`)
- Теги проставляются автоматически при загрузке — проанализируй каждый фильм и назначь теги на основе жанра, года, режиссёра, атмосферы
- Лимит API Кинопоиска — 500 req/day. Если мешает — кэшируй в БД, батчингуй, или назначай теги из метаданных CSV без лишних запросов

### 2. Создать профиль киномана

На основе загруженных данных (оценки, жанры, теги) — сгенерировать файл `kinoclaude/profile.md`.

Профиль должен отражать:
- Любимые жанры и поджанры
- Что точно не заходит
- Любимые режиссёры / актёры
- Характерные паттерны вкуса (например: «любит медленное европейское кино», «ценит атмосферу выше сюжета»)

Профиль статичный, обновляется редко. Генерируется один раз.

### 3. Добавить MCP-инструменты в kinoclaude_mcp.py

Добавь инструменты:

- `get_my_ratings(status=None, genres=None, tags=None, limit=50)` — выборка из БД с фильтрами. **Главный инструмент фильтрации** — Claude дёргает его чтобы не тащить все 900 фильмов в контекст
- `rate_film(film_id, title, rating, status)` — сохранить/обновить оценку
- `get_profile()` — вернуть содержимое `profile.md`
- `add_to_blacklist(film_id, title, reason)` — добавить в чёрный список
- `get_blacklist()` — список нежелательных фильмов

### 4. Написать системный промпт для сценария «Посоветуй фильм»

Файл `kinoclaude/recommend_prompt.md` — инструкция для Claude как вести диалог:

- Читает профиль через `get_profile()`
- Задаёт уточняющие вопросы (не больше 3)
- Формирует фильтры → `get_my_ratings()` чтобы исключить просмотренное
- Ищет кандидатов через `get_films_by_filters()` или `search_films()`
- Обогащает через `get_film()` и веб-поиск
- Убеждает пользователя — апеллирует к его вкусам из профиля, проводит параллели с тем что уже понравилось

## Важно

- **Не упирайся в одну концепцию реализации.** Если подход не работает — ищи другой путь к той же цели
- Лимит Кинопоиска API — 500 запросов в день. Кэшируй данные в БД при первом запросе
- Claude Desktop не поддерживает `type: sse` напрямую — транспорт уже настроен через `npx mcp-remote@latest`, не трогай
- JSON конфиги — только через `json.dump()`, без trailing comma
- Деплой изменений на VPS: `sudo systemctl restart kinoclaude`
