# План внедрения OpenClaw Secretary

## Уже сделано
- OpenClaw живет на `homeserver` и подключен к Telegram.
- В workspace развернуты файлы личности: `AGENTS.md`, `IDENTITY.md`, `USER.md`, `TOOLS.md`, `MEMORY.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`.
- Продуктовая рамка изменена: личный секретарь, а не девопс-мониторинг.
- Холодный тест подтвердил, что агент видит новую идентичность.
- Создана схема таблиц `assistant_*` для PostgreSQL.
- Добавлен MVP-скрипт `scripts/openclaw-cost-footer.py`, который считает рублевый footer из OpenClaw session JSONL без участия модели.

## Фаза 1 — память и контекст
Цель: Сергей может написать “запомни”, “найди”, “разбери”, “что висит”, а бот отвечает из личной и проектной памяти.

Состав:
- таблицы `assistant_*` в PostgreSQL;
- read-only индекс по `brain/`, `projects/`, `infra/`, `sessions/`;
- безопасная запись только в `assistant_*` и дневные markdown-заметки;
- короткие summaries вместо полного контекста.

## Фаза 2 — голос и inbox
Цель: голосовые и хаотичные мысли превращаются в заметки, задачи, решения или планы.

Состав:
- speech-to-text;
- inbox для сырых мыслей;
- команда “разбери inbox”;
- вечерний разбор только по запросу.

## Фаза 3 — напоминания и календарь
Цель: секретарь помогает планировать день/вечер и помнить follow-up.

Состав:
- `assistant_reminders`;
- n8n webhook для доставки напоминаний;
- календарь в режиме read/draft;
- создание событий только после подтверждения.

## Фаза 4 — документы и почта
Цель: читать, пересказывать и готовить черновики.

Состав:
- Nextcloud как хранилище;
- PDF/OCR;
- DocAI для поиска по документам;
- почта read/draft, отправка только confirm.

## Фаза 5 — внешний мир
Цель: актуальные исследования и покупки без лишнего расхода.

Состав:
- web-search только когда локальной памяти мало или тема актуальная;
- summaries источников;
- режимы “быстро” и “исследуй глубоко”.

## Фаза 6 — домашние инструменты
Цель: бытовые и серверные действия по запросу.

Состав:
- Uptime Kuma, Jellyfin, qBittorrent, Immich, Nextcloud;
- Home Assistant в будущем;
- действия только allowlist + confirm.

## Стоимость
- Держать постоянный bootstrap до 5-8k токенов.
- Все крупные знания доставать поиском по запросу.
- Не использовать дорогую модель для классификации простых команд.
- Разделить “быстро ответить” и “подумать глубоко”.
