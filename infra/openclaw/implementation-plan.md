# План внедрения OpenClaw Secretary

## Северная звезда
OpenClaw — личный секретарь Сергея в Telegram: память, разбор мыслей, поиск по личному контексту, напоминания, документы, черновики, решения и аккуратные действия с подтверждением.

Это не девопс-бот. Серверы и домашние сервисы — один из наборов инструментов, а не центр продукта.

## Слой 0 — фундамент и личность ✅
Цель: OpenClaw живет 24/7 на `homeserver`, отвечает в Telegram и понимает свою роль как личного секретаря.

Сделано:
- OpenClaw живет на `homeserver` и подключен к Telegram.
- В workspace развернуты файлы личности: `AGENTS.md`, `IDENTITY.md`, `USER.md`, `TOOLS.md`, `MEMORY.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`.
- Продуктовая рамка изменена: личный секретарь, а не девопс-мониторинг.
- Холодный тест подтвердил, что агент видит новую идентичность.
- Создана схема таблиц `assistant_*` для PostgreSQL.
- Добавлен MVP-скрипт `scripts/openclaw-cost-footer.py`, который считает рублевый footer из OpenClaw session JSONL без участия модели.

Критерий готовности:
- На вопрос “кто ты для Сергея?” агент отвечает как Коготь, личный секретарь.
- В `jarvis_memory` есть таблицы `assistant_notes`, `assistant_tasks`, `assistant_decisions`, `assistant_reminders`, `assistant_user_profile`.
- Cost footer можно посчитать отдельно по session JSONL.

## Слой 1 — память и контекст ⏭️ следующий
Цель: Сергей может написать “запомни”, “найди”, “разбери”, “что висит”, а бот отвечает из личной и проектной памяти.

Состав:
- таблицы `assistant_*` в PostgreSQL;
- read-only индекс по `brain/`, `projects/`, `infra/`, `sessions/`;
- безопасная запись только в `assistant_*` и дневные markdown-заметки;
- короткие summaries вместо полного контекста.

Что надо сделать:
- Подключить `Claude_Home` на `homeserver` read-only или синхронизировать контекст через git.
- Сделать команду/инструмент поиска по `brain/`, `projects/`, `infra/`, `sessions/` без загрузки всего контекста.
- Реализовать сохранение заметок/задач/решений в `assistant_*`.
- Проверить сценарии:
  - “запомни: ...”
  - “найди, что мы решили про OpenClaw”
  - “что у нас висит по проектам”
  - “разбери вот эту мысль: ...”
- Не давать произвольный shell из Telegram.

Критерий готовности:
- OpenClaw может сохранить простую заметку в `assistant_notes`.
- OpenClaw может найти ответ по `brain/`/`projects/` точечным поиском.
- Для обычного вопроса не тянется весь репозиторий в prompt.

## Слой 2 — голос и inbox
Цель: голосовые и хаотичные мысли превращаются в заметки, задачи, решения или планы.

Состав:
- speech-to-text;
- inbox для сырых мыслей;
- команда “разбери inbox”;
- вечерний разбор только по запросу.

## Слой 3 — напоминания и календарь
Цель: секретарь помогает планировать день/вечер и помнить follow-up.

Состав:
- `assistant_reminders`;
- n8n webhook для доставки напоминаний;
- календарь в режиме read/draft;
- создание событий только после подтверждения.

## Слой 4 — документы и почта
Цель: читать, пересказывать и готовить черновики.

Состав:
- Nextcloud как хранилище;
- PDF/OCR;
- DocAI для поиска по документам;
- почта read/draft, отправка только confirm.

## Слой 5 — внешний мир
Цель: актуальные исследования и покупки без лишнего расхода.

Состав:
- web-search только когда локальной памяти мало или тема актуальная;
- summaries источников;
- режимы “быстро” и “исследуй глубоко”.

## Слой 6 — домашние инструменты
Цель: бытовые и серверные действия по запросу.

Состав:
- Uptime Kuma, Jellyfin, qBittorrent, Immich, Nextcloud;
- Home Assistant в будущем;
- действия только allowlist + confirm.

## Сквозной слой — стоимость
- Держать постоянный bootstrap до 5-8k токенов.
- Все крупные знания доставать поиском по запросу.
- Не использовать дорогую модель для классификации простых команд.
- Разделить “быстро ответить” и “подумать глубоко”.
- Стоимость ответа считает внешний слой по реальному `usage.cost.total`, а не модель.
- Курс USD/RUB обновлять редко и кешировать.
- Если usage нет — не выдумывать стоимость.

## Сквозной слой — безопасность
- Read-only по умолчанию.
- Draft перед внешним действием.
- Confirm для записи, отправки, webhooks, календаря, почты и серверных действий.
- Forbidden: печать секретов, произвольный shell из Telegram, удаление файлов, отправка без подтверждения.
