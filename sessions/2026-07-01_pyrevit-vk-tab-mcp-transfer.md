## Часть 2: pyRevit-расширение для инженера ВК
Пользователь попросил создать pyRevit-расширение со своей вкладкой. pyRevit изначально не был
установлен — установка и запуск заняли большую часть сессии из-за серии нетривиальных проблем
(см. `brain/grabli.md`, раздел "pyRevit на рабочем ПК" — там всё подробно, здесь коротко):

1. Битые загрузки инсталлятора (MCP-таймаут обрывал скачивание) → фикс через `Start-BitsTransfer`
2. Ни CLI, ни голый git-клон не поставили рабочие бинарники → сработал только GUI-инсталлятор
3. Не хватало .NET Desktop Runtime 8.0 → доставлен отдельно
4. **Главная нерешённая загадка**: DLL по пути в `AppData\Roaming\pyRevit-Master\...` физически
   существовал и грузился через PowerShell, но Revit настойчиво писал "assembly не существует".
   Причину не нашли (не WDAC, не MOTW, не кодировка, не redirect). **Обход**: перенос всего клона
   в короткий путь `C:\pyRevit-Master` — заработало сразу.
5. Пользовательские расширения (`userextensions`/Custom Extension Directories) на этой машине
   **не работают вообще** — ни через CLI, ни через GUI Settings, никакой диагностики не помогло
   найти причину. **Обход**: расширение положено прямо в `C:\pyRevit-Master\extensions\` —
   папку, которую движок гарантированно сканирует (там же родной `pyRevitCore.extension`).

### Результат
Вкладка **Избранное** (переименована из VK) в Revit 2024, живёт в
`C:\pyRevit-Master\extensions\VK_Tools.extension`, бэкап исходников — `projects/pyrevit-vk/`:
- `Инструменты.panel` — три заглушки (Нумерация, Проверка, Спецификации)
- `Видимость.panel` → **«Скрыть системы»** — рабочий инструмент, не заглушка: находит все
  типы систем трубопроводов/воздуховодов в проекте по параметру `RBS_SYSTEM_ABBREVIATION_PARAM`
  (найдёт реальные В1/В2/К1/К2 и т.п.), чекбоксами скрывает/показывает во всех видах проекта.
  **Не протестирован вживую** (нет доступа к запуску Revit) — код написан по документации API.
