# Revit MCP — 22.04.2026

## Что обсуждали
Подключение Claude Desktop к Autodesk Revit через MCP. Цель — управлять моделью голосом/текстом: чертить оси, стены, перекрытия, потенциально арматуру.

## Что решили
- Используем open-source стек: revit-mcp (github.com/revit-mcp/revit-mcp) + плагин revit-mcp-plugin для Revit
- Python-версия не подходит (требует pyRevit). Коммерческий NoticaTab — платный. Выбрали оригинальный TypeScript-репозиторий
- Кириллица в именах осей не поддерживается плагином — только латиница

## Что сделали
1. Node.js уже был установлен (v24.14.0)
2. Установили Visual Studio Build Tools 2022 с компонентом «Desktop development with C++» — нужно для компиляции better-sqlite3
3. Клонировали репо: `git clone https://github.com/revit-mcp/revit-mcp.git C:\revit-mcp`
4. Собрали: `npm install --include=dev && npm run build` → получили `C:\revit-mcp\build\index.js`
5. Распаковали Revit-плагин в `C:\Users\gor-r\AppData\Roaming\Autodesk\Revit\Addins\2024\`
6. Исправили .addin файл — заменили относительный путь на абсолютный к RevitMCPPlugin.dll
7. Добавили в claude_desktop_config.json запись:
```json
"revit-mcp": {
  "command": "node",
  "args": ["C:\\revit-mcp\\build\\index.js"]
}
```
8. Первая команда: нарисовали разбивочную сетку 5×4 оси, шаг 6000 мм — **успешно**

## Результат
✅ Связка Claude Desktop → revit-mcp MCP Server → Revit 2024 работает  
✅ Оси созданы в модели через текстовую команду  
⚠️ Кириллические имена осей не передаются — имена выходят кодом (AKN, AKO...)
