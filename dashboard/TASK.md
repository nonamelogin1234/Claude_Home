# ЗАДАЧА: Мониторинг-дашборд

## Результат

Развернуть self-hosted дашборд мониторинга, доступный по адресу `monitor.myserver-ai.ru`.

Дашборд должен давать полную картину состояния инфраструктуры:
- Метрики хостов (CPU, RAM, диск, сеть) — VPS и домашний сервер
- Состояние Docker-контейнеров
- Метрики PostgreSQL (БД `jarvis_memory`)
- Статус ключевых systemd-сервисов
- Алерты в Telegram при падении чего-либо

## Что нужно сделать перед реализацией

Самостоятельно изучи контекст инфраструктуры (brain/infra.md, brain/projects.md).
Составь список всего, что нужно мониторить. Если найдёшь сервисы или ресурсы,
не отражённые в контексте — добавь их в мониторинг самостоятельно.

## Моё мнение на реализацию

Я бы взял **Grafana + Prometheus + Node Exporter + cAdvisor + Postgres Exporter + Alertmanager**.
Это индустриальный стандарт, всё поднимается через docker-compose на VPS,
метрики с домашнего сервера тянутся через WireGuard.
Готовые дашборды импортируются по ID: Node Exporter Full — 1860, Docker — 193, PostgreSQL — 9628.

Это моё мнение, не директива. Если видишь более подходящее решение — действуй по своему.
Главное — результат, а не инструмент.

Если зашёл в тупик с выбранным подходом и потратил на него уже достаточно времени —
не трать больше, переключись на альтернативу.
