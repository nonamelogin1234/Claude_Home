# ЗАДАЧА: Miniflux + n8n + Telegram дайджест

## Цель
Поднять Miniflux на VPS, подключить к существующему PostgreSQL, настроить интеграцию с n8n так, чтобы каждое утро в Telegram приходил дайджест непрочитанных статей — краткий, через Claude.

## Что должно получиться
1. Miniflux работает на VPS, доступен через браузер (поддомен или путь)
2. Miniflux использует существующий PostgreSQL (jarvis_memory или отдельная БД)
3. n8n по расписанию забирает непрочитанные статьи через Miniflux API
4. Claude анализирует статьи и пишет краткий дайджест
5. Дайджест летит в Telegram

## Технические детали

### Miniflux
- Docker-контейнер на VPS
- БД: PostgreSQL на VPS (контейнер postgres, IP 172.18.0.4, user: jarvis, pass: jarvis_pass)
- Можно отдельную БД внутри того же PostgreSQL (например `miniflux`)
- Порт: выбери незанятый, пробрось через nginx на поддомен `rss.myserver-ai.ru`

### n8n workflow
- Триггер: Schedule, каждый день в 08:00 МСК
- Шаг 1: HTTP Request → Miniflux API `/v1/entries?status=unread&limit=20`
  - Auth: API Key (создать в Miniflux)
- Шаг 2: отправить список статей (title + url + content) в Claude (HTTP Request → Anthropic API или через AI-ноду)
- Шаг 3: Telegram → отправить дайджест

### Claude промпт для дайджеста
Примерно такой (можешь улучшить):
```
Вот список непрочитанных статей из RSS. 
Сделай краткий дайджест на русском: 
- выдели 3-5 самых интересных
- по каждой 1-2 предложения о чём она
- в конце общая сводка одной строкой
Статьи: {статьи}
```

### После отправки дайджеста
Пометить статьи как прочитанные через Miniflux API: `PUT /v1/entries` с `{"status": "read", "entry_ids": [...]}`

## Что НЕ нужно делать
- Не настраивать подписки — это Сергей сделает сам через веб-интерфейс
- Не усложнять: простой линейный workflow без ветвлений

## Ожидаемый результат сессии
- Miniflux запущен, логин работает
- nginx роутит на rss.myserver-ai.ru
- n8n workflow создан и протестирован (можно вручную запустить)
- В Telegram пришёл тестовый дайджест
