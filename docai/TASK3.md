# DocAI v3 — полноценное приложение с UI и постоянной базой

## Что уже есть (не переделывать)

- `docai.py` — извлечение PDF → JSON, работает хорошо
- `ask.py` — RAG pipeline: build индекса + гибридный поиск (semantic + BM25 + RRF) + ответ через Claude API

Всё это становится внутренностью нового приложения. Логику не трогать — только обернуть.

## Что нужно построить

### Концепция: проекты и файлы

Пользователь работает с **проектами** (папками). В каждом проекте — несколько PDF томов.
Проекты хранятся в постоянной базе данных на ПК. При закрытии/открытии программы всё на месте.

Пример структуры в голове пользователя:
```
Проект "Агалатово"
    АР — Архитектурные решения.pdf
    КР — Конструктивные решения.pdf
    ИОС — Инженерные системы.pdf

Проект "Тентюки"
    АР.pdf
    КР.pdf
```

### Функции приложения

1. **Управление проектами**
   - Создать проект (задать название)
   - Переименовать, удалить проект

2. **Управление файлами в проекте**
   - Добавить PDF в проект (выбрать файл через диалог)
   - Удалить файл из проекта
   - Переместить файл в другой проект
   - Показать статус файла: не обработан / обрабатывается / готов к поиску

3. **Обработка (build индекса)**
   - Кнопка "Обработать" для конкретного файла или всего проекта
   - Прогресс обработки виден в UI
   - Повторная обработка только если файл изменился (по дате/хешу)

4. **Поиск и вопросы**
   - Выбрать проект (или несколько) для поиска
   - Ввести вопрос
   - Получить ответ с указанием источников (том, страница)

5. **Кэш модели embeddings**
   - Модель скачивается один раз, кэшируется локально
   - При повторных запусках — не скачивается заново

### База данных

Постоянная локальная БД. Хранит:
- Проекты (id, название, дата создания)
- Файлы (id, проект, путь к оригинальному PDF, путь к JSON, путь к индексу, статус, хеш файла, дата обработки)
- Настройки (путь к кэшу модели, Anthropic API key если нужно)

Размещение БД и данных — в `%APPDATA%\DocAI\` или рядом с exe, на усмотрение.

### Интерфейс

Десктопное приложение. Выбор технологии — на твоё усмотрение.
Требования: работает на Windows, не требует установки браузера или отдельного сервера.
Варианты: tkinter, PyQt, PySide, customtkinter — что считаешь лучшим.

Минимальный но функциональный UI — не нужно делать красиво, нужно чтобы работало.

### Упаковка в exe

После того как приложение работает — упаковать через PyInstaller в один exe.
Модель embeddings кэшируется в `%APPDATA%\DocAI\models\` — не упаковывать внутрь exe.

## Ограничения

- Всё локально, Windows
- Документация конфиденциальная — никаких внешних сервисов кроме Anthropic API
- Anthropic API key: пользователь вводит в настройках приложения, хранится в БД
