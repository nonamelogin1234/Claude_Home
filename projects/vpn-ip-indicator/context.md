Причина — Kill Switch в настройках AmneziaVPN блокировал весь трафик мимо туннеля, включая LAN. Решение: выключить Kill Switch в настройках AmneziaVPN (сам VPN остаётся включён, IP в индикаторе не меняется). Подробности диагностики — `brain/grabli.md`.

## ГРАБЛИ
- `.vbs`-файл с кириллицей нельзя сохранять с UTF-8 BOM — `cscript` падает с ошибкой "несовместимый тип данных" на первой строке. Кириллицу из `.vbs` лучше не использовать вовсе (как в vacation-timer), BOM не ставить.
- `.ps1` с кириллицей наоборот ТРЕБУЕТ UTF-8 BOM, иначе PowerShell 5 при запуске через `bash`/нестандартную консоль читает кириллицу как кракозябры и падает с `ParserError` на строке с русским текстом.
- Иконка может быть скрыта под стрелочкой "отображать скрытые значки" — перетащить на видимую часть панели вручную (Windows это не даёт сделать программно надёжно).

## ТЕХНИЧЕСКИЕ ДЕТАЛИ
```
# Запустить вручную (без перезагрузки)
cscript //nologo "projects\vpn-ip-indicator\launch-vpn-ip-indicator.vbs"

# Установить в автозагрузку (если ярлык пропал)
powershell -ExecutionPolicy Bypass -File "projects\vpn-ip-indicator\install-autostart.ps1"

# Остановить
Через контекстное меню иконки в трее → "Выход", либо:
Get-Process powershell | Where-Object { $_.MainWindowTitle -eq '' } # найти и Stop-Process по Id вручную
```
