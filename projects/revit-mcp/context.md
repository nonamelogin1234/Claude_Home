# RuRevitMCP — контекст проекта

## Что это

Пользовательский CommandSet для Revit MCP. Команды на C# (.NET 4.8, Revit 2024),
которые Claude вызывает через MCP протокол для автоматизации Revit.

## Архитектура

Использует `mcp-servers-for-revit` (npm пакет) как MCP сервер + плагин в Revit.
Пользовательский CommandSet подключается через `command.json`.

Паттерн каждой команды:
```
Commands/XxxCommand.cs         — ExternalEventCommandBase, точка входа от MCP
Services/XxxEventHandler.cs    — IExternalEventHandler + ManualResetEvent, выполняет Revit API
Models/XxxModels.cs            — Request/Result модели с JsonProperty
```

## Ключевые детали

- Все размеры **снаружи в мм**, внутри конвертируем `mm / 304.8` → футы (внутренние единицы Revit)
- `command.json` — регистрация команд, `assemblyPath`: просто `"RuRevitCommandSet.dll"`
- Сборка: `"C:\Program Files\dotnet\dotnet.exe" build src\RuRevitCommandSet.csproj` → DLL автоматом копируется в папку Addins
- Кириллица в именах осей: буквы по ГОСТ (без Ё, З, Й, О, Ъ, Ы, Ь)

## Следующий шаг

### Задача: зафиксировать алгоритм подсчёта ГБ400 (цель: 83 м³)

**Текущий результат PartUtils:** 92.415 м³ (верная модель, 23 стены)

**Найдены 3 класса ошибок PartUtils:**

#### 1. Части в зоне плиты перекрытия (ВЫШЕ_ПЛИТЫ, Z >= 2920мм) — 3.734 м³
PartUtils разбивает стены на границах плиты перекрытия и создаёт лишние части.
Исключить: части с `pbb.Min.Z >= 2920мм / 304.8`.

#### 2. Стена 1092503 уходит ниже Z=0 — часть объёма в фундаментной зоне (ПЕРЕСЕК_НОЛЬ)
Part1538499: Z=-183→3390мм, V=2.706 м³ (полный солид).
Объём ниже Z=0 (~183мм): приблизительно `2.706 * 183/3573 ≈ 0.14 м³`.
Исключить: клипировать части с Z_min < 0 до Z=0.

#### 3. Двойной счёт — несколько D400 частей с перекрывающимися Z-диапазонами (ПЕРЕКРЫТИЕ)
Несколько стен имеют 2 части D400 с одинаковым Z-диапазоном — они считаются дважды!
Список стен с перекрытием:
```
1212764: Part1538503 (V=3.5182) + Part1538504 (V=0.9691) — оба Z=167→2920мм
1213346: Part1538542 (V=1.4955, Z=167→3390) + Part1538543 (V=0.4955, Z=167→2920)
1264662: Part1538569 (V=0.3779, Z=167→3390) + Part1538791 (V=0.1429, Z=167→2920)
1403569: Part1538574 (V=1.7729, Z=167→2920) + Part1538575 (V=1.3596, Z=167→3390)
1507203: Part1538579 (V=3.5182) + Part1538580 (V=0.9691) — оба Z=167→2920мм
1507232: Part1538615 (V=0.4955) + Part1538616 (V=0.4955) — оба Z=167→2920мм
1507263: Part1538642 (V=0.3779, Z=167→3390) + Part1538793 (V=0.1429, Z=167→2920)
1508097: Part1538647 (V=1.7729, Z=167→2920) + Part1538648 (V=1.3596, Z=167→3390)
```
**Гипотеза причины:** каждая из этих стен граничит с другой стеной — PartUtils создаёт отдельные части для каждого "отсека" (до стыка и после), но они пересекаются геометрически. Нужно либо суммировать части через BooleanOperationsUtils.Union, либо отфильтровать дубли.

**Без ВЫШЕ_ПЛИТЫ:** 88.681 м³ (-3.734)
**Без ВЫШЕ_ПЛИТЫ и ПЕРЕСЕК_НОЛЬ (грубо):** ~85.97 м³

Ещё ~3 м³ — это перекрытие частей. Нужно:
1. Собрать все D400 солиды одной стены, сделать Boolean Union, взять итоговый объём
2. Клипировать по Z (исключить Z>2920мм и Z<0мм)

**Код-шаблон для следующей сессии:**
```csharp
// Для каждой стены: Union всех D400 солидов, затем клипировать по Z=0..2920
// Clipping via BooleanOperationsUtils.ExecuteBooleanOperation(solid, clipBox, Intersect)
// clipBox: XYZ(-1e6, -1e6, 0) до XYZ(1e6, 1e6, 2920/304.8) — прямоугольный экструзион
```

## Параметры create_grid_ru

```json
{
  "vertical_spans": [5680, 7050],        // пролёты между вертикальными осями 1,2,3 (мм)
  "horizontal_spans": [750, 7330, 2660]  // пролёты между горизонтальными осями А,Б,В,Г (мм)
}
```

- `vertical_spans` → оси параллельны Y (вертикальные линии) → цифровые: 1=0, 2=5680, 3=12730
- `horizontal_spans` → оси параллельны X (горизонтальные линии) → буквенные: А=0, Б=750, В=8080, Г=10740
