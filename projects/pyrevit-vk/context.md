# pyRevit-расширение для инженера ВК

## Статус: 🟢 работает (2026-07-01)

## Что это
Собственная вкладка **Избранное** на ленте Revit 2024 (pyRevit-расширение), под задачи инженера ВК
(водоснабжение и канализация). Живёт в `VK_Tools.extension/` (копия в этом репо — бэкап исходников,
рабочая копия физически лежит на машине в `C:\pyRevit-Master\extensions\VK_Tools.extension`, см. "Где физически лежит" ниже).

## Структура
```
VK_Tools.extension/
├── lib/
│   └── vk_systems.py                       — общая логика скрытия систем (см. ниже)
└── Избранное.tab/
    ├── Инструменты.panel/
    │   ├── Нумерация.pushbutton/           — заглушка (forms.alert)
    │   ├── Проверка.pushbutton/            — заглушка
    │   └── Спецификации.pushbutton/        — заглушка
    ├── Видимость.panel/
    │   ├── Вода1.stack/  — Скрыть В1 / Скрыть В2 / Скрыть Т3  (кнопки-переключатели)
    │   ├── Вода2.stack/  — Скрыть Т4 / Скрыть К1 / Скрыть К2  (кнопки-переключатели)
    │   └── Скрыть К3.pushbutton/           — кнопка-переключатель (одиночная)
    └── Сантехника.panel/                    — дублёры нативных инструментов Revit
        (вкладка «Системы» → панель «Сантехника и трубопроводы», ПОЛНОСТЬЮ скопирована)
        Труба / Труба в одну линию / Параллельные трубы /
        Соединительные детали трубопроводов / Арматура трубопроводов /
```

**«Скрыть В1/В2/Т3/Т4/К1/К2/К3»** (2026-07-01, переписано дважды с нуля): 7 кнопок-
переключателей прямо на ленте — БЕЗ диалогов и окон. Клик по кнопке сразу
скрывает/показывает соответствующую систему во всех видах проекта; состояние (скрыто/
видно) отражается **текстом кнопки** — галочка ☑/☐ перед аббревиатурой (например,
«☑ В1»), меняется через `script.get_button().set_title(...)`.

**Важно (грабли №1, версия с `script.toggle_icon()`/`on.png`/`off.png` — НЕ РАБОТАЛА):**
первая версия меняла иконку кнопки в рантайме через `script.toggle_icon()`. Пользователь
вживую увидел вместо чекбокса дефолтную generic-иконку pyRevit — это известный баг
pyRevit (`toggle_icon` ломает размер/рендер иконки при вызове, задокументирован на
форуме pyRevitLabs). Заменено на статичную иконку (никогда не меняется в рантайме) +
текст кнопки для состояния — надёжно, проверено через исходники pyRevit
(`pyrevitlib/pyrevit/coreutils/ribbon.py`, метод `_PyRevitRibbonButton.set_title`
напрямую пишет `ItemText` нативной кнопки).

**Важно (грабли №2 — всплывающее окно pyRevit output):** pyRevit по умолчанию
открывает окно-консоль (output) при **каждом** запуске скрипта, даже если скрипт
ничего не печатает — это не баг, а поведение по умолчанию. Пользователь увидел пустое
окно «Скрыть В1» при клике. Исправлено: **каждый** script.py (все 16 кнопок — 7
переключателей + 9 копий инструментов) начинается с `script.get_output().close()`.
Если добавляешь новую кнопку — не забыть эту строку, иначе окно снова вылезет.

Важное исправление по сравнению со старой версией: раньше фильтровали по параметру
`RBS_SYSTEM_ABBREVIATION_PARAM` у типа системы — в этом проекте он **пустой**, поэтому
старый инструмент ничего не находил («не работал»). Проверено вживую через revit-mcp
(`send_code_to_revit`/`ai_element_filter`): реальные типы систем в проекте называются
`ADSK_Водопровод_В1`, `ADSK_Водопровод_В2`, `ADSK_Водопровод_Т3`, `ADSK_Водопровод_Т4`,
`ADSK_Канализация_К1`, `ADSK_Канализация_К2`, `ADSK_Канализация_К3` (категория
«Трубопроводные системы», класс `PipingSystemType`). Новая логика в `lib/vk_systems.py`
ищет типы систем по точному **имени типа** (Name), а элементы — по параметру
`RBS_SYSTEM_TYPE_PARAM` (как и раньше, эта часть была рабочей). Протестировано только
через API-запросы (чтение структуры проекта), реальный клик по кнопке в интерфейсе
Revit пользователь ещё не проверял.

**Панель «Сантехника»**: вместо диалога/копирования кода — каждая кнопка вызывает
`RevitCommandId.LookupCommandId("ID_...")` + `revit.uiapp.PostCommand(...)`, то есть
буквально запускает ту же нативную команду Revit, что и клик на вкладке «Системы».
ID команд получены вживую через `send_code_to_revit` (перебор `Autodesk.Windows.
ComponentManager.Ribbon` → вкладка «Системы» → панель «Сантехника и трубопроводы»).
**Иконки — тоже настоящие, не нарисованные**: вытащены вживую из нативных кнопок
Revit через тот же `send_code_to_revit` (`RibbonButton.LargeImage` → `PngBitmapEncoder`
→ base64 → сохранено как `icon.png`), не generic-заглушка на все кнопки.

**Технические ограничения Revit Ribbon API (важно при развитии дальше):**
- Нативных чекбоксов на ленте не существует — состояние показываем через текст кнопки
  (☑/☐ + аббревиатура), а не через иконку (см. грабли №1 выше).
- Кнопки нельзя расположить в произвольную сетку «2 столбца» — компоновка (stack из
  максимум 3 кнопок в столбец) единственный доступный инструмент, реальное количество
  видимых столбцов дальше зависит от ширины панели/окна Revit у пользователя.
- Новую ПАНЕЛЬ в существующей вкладке Reload в pyRevit подхватывает без перезапуска
  Revit; новую вкладку верхнего уровня — нет (см. правило из "Как добавлять новые
  инструменты" ниже).
- `script.toggle_icon()` не использовать для toggle-кнопок — баг, ломает иконку (см.
  грабли №1). Для смены состояния кнопки использовать `script.get_button().set_title()`.
- Каждый script.py должен начинаться с `script.get_output().close()`, иначе при каждом
  клике вылезает пустое окно-консоль pyRevit (см. грабли №2).

## Где физически лежит и почему НЕ в стандартном месте
pyRevit CLI/GUI по умолчанию сканирует пользовательские расширения из
`%APPDATA%\pyRevit\Extensions` (механизм `userextensions` в конфиге). **Этот механизм на
этой машине не работает** — расширение там просто не подхватывается, никакая
диагностика (логи, кэш, конфиг) не показала причину, при том что путь прописан
и через CLI, и через GUI Settings → Custom Extension Directories.

Рабочий обходной путь: расширение лежит **прямо внутри клона pyRevit**, в
`C:\pyRevit-Master\extensions\VK_Tools.extension` — это `EXTENSIONS_DEFAULT_DIR`
(папка, которую движок гарантированно сканирует, там же лежит `pyRevitCore.extension`
самого pyRevit). Подробности расследования — в `brain/grabli.md`.

**Важно:** если этот путь когда-нибудь понадобится подчистить/переустановить pyRevit —
не потерять `VK_Tools.extension` из `C:\pyRevit-Master\extensions\`, копия в этом
репо — просто бэкап исходников, автоматически туда не разворачивается.

## Как добавлять новые инструменты
1. `ИмяПанели.panel/ИмяКнопки.pushbutton/` внутри `Избранное.tab`
   (в `C:\pyRevit-Master\extensions\VK_Tools.extension\Избранное.tab\`)
2. `script.py` (IronPython, `from pyrevit import revit, DB, forms`) + `icon.png` (32×32, валидный PNG)
3. **Reload** на ленте pyRevit — добавляет новые кнопки в СУЩЕСТВУЮЩИЕ вкладки без перезапуска
4. Новая вкладка верхнего уровня (новый `.tab`) — Revit **надо перезапустить полностью**,
   Reload новые вкладки не создаёт (ограничение Revit Ribbon API)
5. Не забыть скопировать изменения также в `projects/pyrevit-vk/VK_Tools.extension/` в этом
   репо, чтобы не потерять при переустановке

## revit-mcp — есть живой доступ к API
На рабочем ПК подключён MCP `revit-mcp` (в списке инструментов `mcp__revit-mcp__*`),
работает только когда Revit открыт с проектом. Полезные вызовы, которыми уже
пользовались для реальной отладки:
- `say_hello` — проверка связи
- `ai_element_filter` (`data.filterCategory`, напр. `"OST_PipingSystem"`) — вытащить
  элементы категории с параметрами
- `send_code_to_revit` — выполнить произвольный C# в контексте документа; **переменная
  документа называется `document`** (НЕ `doc`, НЕ `Document`, НЕ `UiApp`/`uidoc` — это
  единственное, что реально существует в скоупе шаблона, проверено методом перебора).
  Через `Autodesk.Windows.ComponentManager.Ribbon` можно вживую читать нативные панели
  ленты Revit (вкладки/панели/кнопки/ID команд) — так был найден список кнопок панели
  «Сантехника и трубопроводы» и их `RevitCommandId`.
