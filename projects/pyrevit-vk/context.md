# pyRevit-расширение для инженера ВК

## Статус: 🟢 работает (2026-07-01)

## Что это
Собственная вкладка **Избранное** на ленте Revit 2024 (pyRevit-расширение), под задачи инженера ВК
(водоснабжение и канализация). Живёт в `VK_Tools.extension/` (копия в этом репо — бэкап исходников,
рабочая копия физически лежит на машине в `C:\pyRevit-Master\extensions\VK_Tools.extension`, см. "Где физически лежит" ниже).

## Структура
```
VK_Tools.extension/
└── Избранное.tab/
    ├── Инструменты.panel/
    │   ├── Нумерация.pushbutton/       — заглушка (forms.alert)
    │   ├── Проверка.pushbutton/        — заглушка
    │   └── Спецификации.pushbutton/    — заглушка
    └── Видимость.panel/
        └── Скрыть системы.pushbutton/  — РАБОЧИЙ инструмент (не заглушка)
```

**«Скрыть системы»**: собирает все типы систем трубопроводов/воздуховодов из проекта
(по параметру `RBS_SYSTEM_ABBREVIATION_PARAM`, т.е. найдёт реальные В1/В2/К1/К2 и т.д. из
текущего проекта, а не хардкод), показывает чекбоксы, отмеченные — скрывает во всех
видах проекта (`view.HideElements`), неотмеченные — показывает (`view.UnhideElements`).
Написан по документации Revit API, **не протестирован вживую** (у меня как у Claude нет
доступа к запуску Revit) — при первом реальном использовании может понадобиться правка.

## Где физически лежит и почему НЕ в стандартном месте
pyRevit CLI/GUI по умолчанию сканирует пользовательские расширения из
`%APPDATA%\pyRevit\Extensions` (механизм `userextensions` в конфиге). **Этот механизм на
этой машине не работает** — расширение там просто не подхватывается, никакая
диагностика (логи, кэш, конфиг) не показала причину, при том что путь прописан
и через CLI, и через GUI Settings → Custom Extension Directories.

Рабочий обходной путь: расширение лежит **прямо внутри клона pyRevit**, в
`C:\pyRevit-Master\extensions\VK_Tools.extension` — это `EXTENSIONS_DEFAULT_DIR`
(папка, которую движок гарантированно сканирует, там же лежит `pyRevitCore.extension`
самого pyRevit). Подробности расследования — в `brain/grabli.md`.

**Важно:** если этот путь когда-нибудь понадобится подчистить/переустановить pyRevit —
не потерять `VK_Tools.extension` из `C:\pyRevit-Master\extensions\`, копия в этом
репо — просто бэкап исходников, автоматически туда не разворачивается.

## pyRevit — как установлен на этом компьютере (рабочий ПК)
- Clone: `master_c`, путь `C:\pyRevit-Master` (короткий путь — длинный
  `C:\Users\gor-r\AppData\Roaming\pyRevit-Master\...` не давал Revit загрузить
  `pyRevitLoader.dll`, причина не найдена, см. grabli.md)
- Attach: Revit 2024.2, engine DEFAULT (2712, IronPython)
- pyRevit CLI: `C:\Users\gor-r\AppData\Local\Programs\pyRevit CLI\bin\pyrevit.exe`
- Требует .NET Desktop Runtime 8.0 (доставлен отдельно, `windowsdesktop-runtime-8.0-x64.exe`)
- Конфиг: `C:\Users\gor-r\AppData\Roaming\pyRevit\pyRevit_config.ini`
  (`filelogging=true`, `debug=true`, `bincache=false` — включено для отладки, можно
  вернуть `bincache=true` для скорости, когда всё стабильно)

## Как добавлять новые инструменты
1. `ИмяПанели.panel/ИмяКнопки.pushbutton/` внутри `Избранное.tab`
   (в `C:\pyRevit-Master\extensions\VK_Tools.extension\Избранное.tab\`)
2. `script.py` (IronPython, `from pyrevit import revit, DB, forms`) + `icon.png` (32×32, валидный PNG)
3. **Reload** на ленте pyRevit — добавляет новые кнопки в СУЩЕСТВУЮЩИЕ вкладки без перезапуска
4. Новая вкладка верхнего уровня (новый `.tab`) — Revit **надо перезапустить полностью**,
   Reload новые вкладки не создаёт (ограничение Revit Ribbon API)
5. Не забыть скопировать изменения также в `projects/pyrevit-vk/VK_Tools.extension/` в этом
   репо, чтобы не потерять при переустановке
