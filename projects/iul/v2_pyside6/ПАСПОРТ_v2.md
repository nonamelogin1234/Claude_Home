## 1. Что за программа

**ИУЛ** — генератор «Информационно-удостоверяющих листов» для проектной документации.

- **Вход:** PDF-файлы с именами вида `Раздел ПД №3 АР часть №4.pdf`
- **Выход:** заполненный Excel (.xlsx) + PDF на каждый входной файл

Шаги обработки одного файла:
1. Парсинг имени → шифр раздела, номера раздела/подраздела/части/книги
2. CRC32, размер, дата модификации файла
3. Копирование шаблона Excel в папку результата
4. Excel COM: замена плейсхолдеров `[ИмяПоля]` реальными значениями
5. Excel COM: вставка PNG-подписей в ячейки `[Роль_подпись]`
6. Excel COM: экспорт PDF через `ExportAsFixedFormat`

### Worker thread (строки 603–654)

```python
class Worker(QThread):
    log_signal      = Signal(str, str)   # level, message → ConsolePanel
    progress_signal = Signal(int, int)   # current, total → global progress bar
    row_status      = Signal(int, str)   # file_index, status → FileRowWidget
    done_signal     = Signal(int, int, int)  # ok, failed, warnings
```

COM (`pythoncom.CoInitialize`) вызывается **внутри `Worker.run()`**, не в главном треде. Никогда не трогать этот порядок.

## 6. Критические инварианты — НЕЛЬЗЯ НАРУШАТЬ

1. **`ExcelSession.__exit__` — двойной `try/finally`.**  
   `gc.collect()` и `pythoncom.CoUninitialize()` должны выполняться всегда. Не упрощай.

2. **COM только в Worker.run().**  
   `pythoncom.CoInitialize()` вызывается внутри `ExcelSession.__enter__`, который вызывается внутри `Worker.run()`. Никогда не вызывать из главного треда Qt.

3. **`init_user_data()` вызывается на уровне модуля до `load_config()`.**  
   Порядок: сначала создать файлы в USER_DATA_DIR, потом загрузить их.

4. **`parse_file_name`, `build_values`, `export_one` — чистые функции.**  
   Не тащить туда ссылки на UI-классы.

5. **Плейсхолдеры в шаблоне Excel.**  
   `CRITICAL_TEMPLATE_PLACEHOLDERS` — если хоть один не найден в шаблоне, программа выдаёт предупреждение. При правке шаблона синхронно обновляй этот set.

## 8. История исправленных багов в v2

### Сессия 1 (создание v2)
- Написан `iul_pyside6.py` с нуля на базе `iul_prototype.py`
- Создана резервная копия `v1_tkinter/`
- Написан `ИУЛ_v2.spec`

### Сессия 2 (первые исправления)
- **AttributeError: QPalette.Window** — в PySide6 6.x нужно `QPalette.ColorRole.Window`. Исправлено.
- **DropZone.set_compact ломала окно** — метод менял только высоту, не скрывал дочерние виджеты. Исправлено: добавлен список `_full_widgets`, в compact-режиме они скрываются и показывается `_compact_btn`.
- **SettingsDialog не открывалась** — `Qt.FramelessWindowHint` + `WA_TranslucentBackground` на QDialog не работает надёжно на Windows. Убраны, диалог стал обычным `Qt.Dialog` со стилем через setStyleSheet.
- **FileRowWidget.set_status** — `replaceWidget` работал ненадёжно. Заменён на простое обновление текста и стиля существующего QLabel.
