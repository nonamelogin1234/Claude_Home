# ИУЛ — Информационно-удостоверяющий лист

## ЦЕЛЬ

Desktop-приложение для Windows. Автоматически генерирует ИУЛ (информационно-удостоверяющие листы) для проектной документации: читает имена PDF-файлов → заполняет Excel-шаблон → вставляет подписи → экспортирует в PDF через Microsoft Excel COM.

## СТАТУС

🟡 В процессе — v2 (PySide6) отменена, в приоритете первая версия на Tkinter

## ЧТО СДЕЛАНО

- v1 на Tkinter — рабочий прототип, скомпилирован в .exe (в `_архив/`), снова основная версия
- v2 на PySide6 — переписан с нуля, но отменён и не является приоритетом
- Разбор имён файлов по шаблону (раздел, часть, книга, подраздел)
- Заполнение Excel-шаблона через COM + вставка подписей (PNG → ячейка)
- Пакетная обработка через один экземпляр Excel
- Фоновый поток для обработки (UI не зависает)
- Журнал обработки в файл (YYYYMMDD_HHMMSS.txt)
- config.json — все данные проекта (шифр, разделы, ФИО, роли) вынесены из кода

## СЛЕДУЮЩИЙ ШАГ

Вернуться к v1: проверить текущий exe/исходник, зафиксировать что именно в нём устраивает, и делать дальнейшие правки поверх v1.

## ГРАБЛИ

- Excel COM нестабилен если в системе уже висит фоновый EXCEL.EXE → закрывать перед запуском
- `pythoncom.CoInitialize()` нужен в worker-thread (COM в потоке)
- config.json переопределяет дефолты в коде — специально, чтоб работало и без конфига
- В exe упаковывать только по отдельной команде — пользователь сказал явно

## ТЕХНИЧЕСКИЕ ДЕТАЛИ

```
v1_tkinter/              ← основная версия
v2_pyside6/              ← отменённая версия, не развивать без отдельного решения
```

**Стек v1:** Python, Tkinter, win32com (Excel COM), openpyxl, PyInstaller

**Запуск (dev):**
```
cd v1_tkinter
python iul_prototype.py
```

**Требования:** Microsoft Excel должен быть установлен (нужен COM)

**Имена файлов — формат:**
```
Раздел ПД № 3 АР часть № 4.pdf
Раздел ПД № 5 подраздел ПД № 1 ИОС1 часть № 1 книга № 3.pdf
```
Разделы: ПЗУ, АР, КР, ИОС. Плейсхолдеры в шаблоне: `[Номер_раздела]`, `[Шифр_тома]`, `[CRC_сумма]`, `[Дата_последнего_изменения]` и др.
