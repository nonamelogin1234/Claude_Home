## 1. Что это за программа

Генерирует «Информационно-удостоверяющие листы» (ИУЛ) для проектной документации.

- **Вход:** PDF-файлы с именами вида `Раздел ПД №3 АР часть №4.pdf`
- **Выход:** заполненный Excel (.xlsx) + PDF на каждый входной файл

Шаги обработки для каждого файла:
1. Парсинг имени → шифр раздела, номера раздела / части / книги
2. CRC32, размер и дата изменения файла
3. Копирование шаблона Excel в папку результата
4. Excel COM: замена плейсхолдеров `[ИмяПоля]` реальными значениями
5. Excel COM: вставка PNG-подписей в ячейки `[Роль_подпись]`
6. Excel COM: экспорт PDF через `ExportAsFixedFormat`

## 4. Архитектура путей (строки 29–58)

```python
if getattr(sys, "frozen", False):        # exe-режим
    _BUNDLE_DIR   = Path(sys._MEIPASS)   # ресурсы внутри exe
    USER_DATA_DIR = Path(os.environ["APPDATA"]) / "ИУЛ"
else:                                    # dev-режим
    _BUNDLE_DIR   = Path(__file__).parent
    USER_DATA_DIR = _BUNDLE_DIR          # данные прямо в папке проекта

LOG_DIR        = USER_DATA_DIR / "Журналы"
CONFIG_PATH    = USER_DATA_DIR / "config.json"
TEMPLATE_PATH  = USER_DATA_DIR / "ИУЛ_шаблон.xlsx"
SIGNATURES_DIR = USER_DATA_DIR / "Подписи"
```

`init_user_data()` при старте создаёт `USER_DATA_DIR`, копирует файлы из бандла, **только если их там ещё нет**.

## 5. Парсинг имени файла

Функция `parse_file_name(path)` → возвращает `ParsedName`:

| Поле | Что ищет в имени |
|------|-----------------|
| `section_code` | АР, КР, ПЗУ, ИОС |
| `section_number` | `раздел ПД №N` (если не найден — из `SECTION_NUMBERS[code]`) |
| `subsection_number` | `подраздел ПД №N` |
| `part_number` | `часть №N` |
| `book_number` | `книга №N` |

`ALLOWED_SECTION_CODES = {"ПЗУ", "АР", "КР", "ИОС"}` (строка 66)

Из `ParsedName` строятся:
- **Шифр тома:** `{PROJECT_CODE}-{section_code}.{part}.{book}` → `1605-2022-АР.4`
- **Номер тома:** `{section_number}.{part}.{book}` → `3.4`

## 8. Критические инварианты — НЕЛЬЗЯ НАРУШАТЬ

1. **`ExcelSession` — двойной `try/finally` в `__exit__`.** `gc.collect()` и `CoUninitialize()` должны выполняться всегда. Не упрощай.

2. **`process_ui_queue` — оба условия в reschedule (строка 1176):**
   `if self.worker_thread and (self.worker_thread.is_alive() or not self.ui_queue.empty())`
   Оба обязательны — иначе кнопка «Создать ИУЛ» может остаться задизейбленной.

3. **`pythoncom.CoInitialize()` вызывается в worker-треде** (внутри `ExcelSession.__enter__`), не в main-треде. COM нельзя передавать между тредами.

4. **Парсинг отделён от GUI.** `parse_file_name`, `build_values`, `export_one` — чистые функции без ссылок на Tk.

5. **`init_user_data()` вызывается на уровне модуля** до `load_config()`. Порядок важен.

6. **`CRITICAL_TEMPLATE_PLACEHOLDERS`** — программа выдаёт предупреждение если хоть один не найден. При правке шаблона синхронно обновляй этот set.

## 9. Excel COM — ключевые особенности

- `win32com.client.DispatchEx` — изолированный процесс Excel (не присоединяется к открытому)
- `Visible = False` — Excel невидим пользователю
- Один экземпляр Excel открывается **на весь батч** — это намного быстрее
- `ExportAsFixedFormat(0, path)` — экспорт в PDF без диалогов

## 10. Как добавить новый раздел (например, ТХ)

1. В `iul_prototype.py`:
   - Строка 66: `ALLOWED_SECTION_CODES` — добавь `"ТХ"`
   - Строки 72–78: `SECTION_NUMBERS` — `"ТХ": "6"`
   - Строки 80–88: `SECTION_NAMES` — `"ТХ": "Технологические решения"`
   - Строки 90–101: `PART_NAMES` — части для `("ТХ", "1")` и т.д.
   - Строки 109–134: `PEOPLE` — `"ТХ": "Фамилия И.О."` для каждой роли
   - Строки 136–141: `PROJECT_ROLE` — `"ТХ": "Главный инженер проекта"`
   - `SettingsDialog._SECTIONS` (строка 742): добавь `"ТХ"`
2. В `config.json`: добавь записи во все словари
3. Пересобери exe
