# Передача проекта ИУЛ

## Назначение
Прототип создает информационно-удостоверяющие листы по файлам пользователя:
- читает данные из имени исходного файла;
- заполняет `ИУЛ_шаблон.xlsx`;
- вставляет подписи из папки `Подписи`;
- сохраняет PDF в выбранную папку;
- сохраняет редактируемый Excel в подпапку `ИУЛ_в_ред_формате`;
- пишет подробный журнал обработки в `ИУЛ_журнал_YYYYMMDD_HHMMSS.txt`.

## Основные файлы
- `iul_prototype.py` - основной прототип с GUI на Tkinter и экспортом через Microsoft Excel COM.
- `config.json` - конфиг проекта: шифр проекта, номера/названия разделов, части, книги, ФИО, роли.
- `ИУЛ_шаблон.xlsx` - Excel-шаблон ИУЛ.
- `Подписи/*.png` - картинки подписей.
- `Запустить_прототип.bat` - запуск прототипа двойным кликом.
- `Задача.doc` - исходное ТЗ.
- `Корректировки.doc` - дополнительные корректировки от пользователя.
- `Результаты_теста` - папка с последними пользовательскими тестами.

## Как запускать
1. Убедиться, что установлен Microsoft Excel.
2. В папке проекта запустить `Запустить_прототип.bat`.
3. Добавить исходные файлы, выбрать папку результата, нажать `Создать ИУЛ`.

## Приоритетная проверка следующему разработчику
1. Запустить `Запустить_прототип.bat` и проверить UI после перевода на поток.
2. Проверить, что `pythoncom.CoInitialize()` корректно работает в worker-thread.
3. Проверить, что сообщения из worker-thread безопасно доходят в Tkinter через очередь.
4. Проверить, что `ИУЛ_шаблон.xlsx` содержит только `[Дата_последнего_изменения]`, без старого пробела.
5. Проверить генерацию на файлах:
   - `Раздел ПД  № 3 АР часть №4.pdf`
   - `Раздел ПД № 5 подраздел ПД № 1 ИОС1 часть № 1 книга № 3.pdf`
6. Проверить, что в `D5` получается `Часть 4`, `Книга 3`, с пробелами.
7. Решить, оставлять ли совместимость со старым `[Дата_последнего _изменения]` в коде или удалить после стабилизации шаблона.

## Известные риски
- `config.json` уже добавлен, но код все еще содержит дефолтные словари. Это сделано специально: программа работает без конфига, а конфиг переопределяет значения.
- Фоновая обработка добавлена недавно и требует ручного теста.
- Excel COM может вести себя нестабильно, если в системе уже висит старый процесс EXCEL. Перед тестом лучше закрыть старые фоновые Excel-процессы.
- Специальная логика строки 2 заменена с координат `A2` на поиск набора плейсхолдеров в ячейке, но это тоже надо проверить на шаблоне после будущих правок.
