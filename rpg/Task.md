# 🎮 RPG-трекер прогресса Сергея — Техническое задание для Claude Code

## Контекст и философия проекта

Прочитай этот документ **целиком** перед тем как писать хоть строчку кода.

Сергей — 27 лет, Санкт-Петербург. Начал трансформацию тела: 117 кг → цель ~85 кг, зал 3×/неделю, фуллбади программа.
Вся активность уже пишется в PostgreSQL на VPS: тренировки из Hevy, вес из Picooc, сон и шаги из Mi Band через Health Connect.

Задача: сделать красивый живой дашборд в стиле DnD / тёмное фэнтези, который **сам читает эти данные** и отображает
прогресс как RPG-путешествие героя. Никакого ручного ввода. Всё автоматически.

Это не MVP и не «хоть что-то работающее». Это **полноценный продукт** с красивым UI, живыми данными и рабочей логикой.

## Экраны и компоненты

### 1. Карточка героя (верхний главный блок)

- Имя: **Сергей**
- Класс: **Воин — Путь Трансформации**
- Уровень рассчитывается по количеству тренировок:
  - 1–10 тренировок = Уровень 1 (Новобранец)
  - 11–25 = Уровень 2 (Воин)
  - 26–50 = Уровень 3 (Ветеран)
  - 51–100 = Уровень 4 (Чемпион)
  - 100+ = Уровень 5 (Легенда)
- Прогресс-бар к следующему уровню (% тренировок до порога)

**Характеристики (из реальных данных):**

| Характеристика | Источник | Формула |
|---|---|---|
| Сила | hevy_sets | Макс. weight_kg в жиме ногами на последней тренировке |
| Выносливость | health_daily_summary | Среднее steps за 7 дней / 100 |
| Восстановление | health_daily_summary | Средний deep_sleep_min за 7 дней |
| Объём | hevy_workouts | Сумма total_volume_kg за 30 дней |

---

### 2. Квест-лог

**Активные квесты** — определяются автоматически на основе данных:

- **«Воин без пропусков»** — серия тренировок без перерыва >4 дней (считать consecutive workout weeks)
- **«Страж сна»** — 5 ночей подряд с deep_sleep_min > 120
- **«Покоритель объёма»** — побить рекорд недельного total_volume_kg
- **«Верный путь»** — 10 тренировок без пропуска целой недели

**Завершённые квесты** (архив, если условие выполнено):

- «Первый шаг» — есть хотя бы 1 тренировка ✓
- «Ветеран зала» — 10+ тренировок ✓
- «Железный человек» — 25+ тренировок ✓

Каждый квест: иконка (emoji), название, описание условия, статус (активен / выполнен / заблокирован).

---

### 3. Боссы — контрольные точки по весу

Список боссов в порядке убывания сложности:

| Босс | Цель (кг) | Статус |
|---|---|---|
| Страж Порога | 110 | побеждён / нет |
| Хранитель Плато | 105 | побеждён / нет |
| Дракон Сотни | 99 | побеждён / нет |
| Теневой Лорд | 90 | побеждён / нет |
| Финальный Босс | 85 | побеждён / нет |

Для каждого босса:
- Иконка/символ в RPG-стиле
- Прогресс-бар от текущего веса к цели
- Статус: 🗡️ Активный (текущий босс) / ✅ Побеждён / 🔒 Недосягаем пока


---

### 4. Блок статистики (карточки)

- Вес сегодня (последний замер из body_measurements)
- Динамика веса: мини-график последних 30 дней (Chart.js, линейный)
- Всего тренировок
- Тренировок за последние 30 дней
- Текущая серия (сколько недель подряд есть хотя бы 1 тренировка)
- Суммарный объём за всё время (кг)
- Рекордная тренировка по объёму (дата + значение)
- Средний объём за последние 5 тренировок

---

### 5. Лог событий (правая колонка или нижняя панель)

Последние 10–15 событий в хронологическом порядке, стиль RPG-лога:

```
[31.03] ⚔️  Воин завершил тренировку. Объём: 7476 кг (+6.5% к прошлой)
[31.03] ⚖️  Вес зафиксирован: 117.0 кг
[30.03] 🌙  Ночь отдыха. Глубокий сон: 143 мин
[28.03] 🏆  Новый рекорд: горизонтальная тяга блока — 33.5 кг
[25.03] ⚔️  Тренировка завершена. Объём: 6202 кг
```

Источники событий:
- hevy_workouts → события тренировок
- body_measurements → события взвешивания
- health_daily_summary → события сна (если deep_sleep > 120 мин — позитивное событие)
- Рекорды по упражнениям (сравниваем каждый сет с историческим максимумом)

Стиль: тёмный фон, золотистый текст, мелкий моноширинный шрифт, прокручиваемый список.

## API эндпоинты (FastAPI)

```
GET /api/hero          → карточка героя (уровень, характеристики, прогресс)
GET /api/quests        → список квестов (активные + завершённые)
GET /api/bosses        → список боссов со статусами и прогресс-барами
GET /api/stats         → общая статистика (карточки)
GET /api/events        → лог последних событий (15 штук)
GET /api/weight-chart  → данные для графика веса [{date, weight}, ...]
```

Все эндпоинты:
- Возвращают JSON
- Кэшируются in-memory на 5 минут (простой dict + timestamp)
- При ошибке БД возвращают последние закэшированные данные или понятную заглушку

## Технические требования к фронтенду

- **Автообновление**: `setInterval` каждые 5 минут — перезапрашивает все эндпоинты и обновляет UI без перезагрузки страницы
- **Responsive**: корректно отображается на десктопе (1920px) и телефоне (375px)
- **Анимации**: плавное появление карточек при загрузке (CSS transitions), прогресс-бары анимируются при обновлении
- **Состояния загрузки**: скелетон или спиннер пока данные не пришли
- **Состояние ошибки**: при недоступности API — показать последние данные с пометкой «обновлено X минут назад»
- Chart.js подключать через CDN
