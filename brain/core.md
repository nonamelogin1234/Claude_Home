# ЯДРО — ОПРЕДЕЛЕНИЕ ОКРУЖЕНИЯ

> Первым делом определи, на каком компьютере мы работаем, и прочитай соответствующий файл.

## Как определить окружение

Проверь путь репозитория или `$env:USERNAME` / `whoami`:

| Признак | Компьютер | Файл с ядром |
|---|---|---|
| Репо в `C:\Users\torganov-a\Documents\Claude_Home`, user `torganov-a` | Рабочий ПК | `brain/core-work.md` |
| Репо в `C:\Users\no-na\Desktop\2027\Codex_Home`, user `no-na` | Домашний ПК | `brain/core-home.md` |
| Repo на `/home/sergei/...` или работаем по SSH с homeserver | homeserver | см. `brain/workstations.md` → раздел "homeserver" |
| Работаем напрямую на VPS (`147.45.238.120`) | VPS | см. `brain/workstations.md` → раздел "VPS" |

Если ни один признак не подошёл (новый компьютер) — открыть `brain/workstations.md`, свериться с таблицей "Окружения" и завести новый `core-*.md` по образцу существующих.

## Дальше

1. Прочитать нужный `brain/core-work.md` или `brain/core-home.md` целиком — там пути, команды к серверам, MCP этого компьютера, структура репо.
2. Дать одну строку подтверждения: «Прочитано: core-work/core-home + [проект если есть]».

## Общее для всех окружений

- `brain/grabli.md` — только дополнять снизу, **никогда не удалять**
- Перед нетривиальным кодом (>2 файлов, новый сервис) — обсудить архитектуру
- Настроили MCP/инструмент/ключ → зафиксировать в `brain/workstations.md`
- Секреты не в коде и не в репо
- После значимой задачи — коммит
- Пути в файлах — всегда относительные от корня репо
- Не путать инструкции одного компьютера с другим — команды, пути и секреты разные
