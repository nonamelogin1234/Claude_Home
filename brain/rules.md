# ПРАВИЛА РАБОТЫ

## Структура репозитория

```
Claude_Home/
├── brain/           ← системные знания (загружается в каждой сессии)
├── projects/        ← код-проекты: есть код, ведётся разработка
├── infra/           ← инфраструктура: конфиги, сервисы, документация
├── archive/         ← замороженные/завершённые проекты
└── sessions/        ← саммари важных сессий claude.ai
```

**projects/** — когда здесь:
- Есть исходный код (Python, C#, JS и т.д.)
- Клод активно помогает с разработкой
- Примеры: kinoclaude, docai, rpg-tracker, revit-mcp

**infra/** — когда здесь:
- Конфиги, docker-compose, systemd-сервисы
- Документация работающих сервисов без активной разработки
- Примеры: homepage, vpn-hide, navidrome, dashboard

**archive/** — когда здесь:
- Проект заморожен или заменён новой версией
- Код сохранён для справки

## Файлы каждого проекта

**Обязательно:**
- `context.md` — цель, статус, что сделано, следующий шаг, грабли, технические детали

**Опционально (только если работаем в Claude Code):**
- `CLAUDE.md` — автоматически загружается Claude Code при открытии папки

**Не нужно:**
- TASK.md, TASK2.md и т.д. — это мусор прошлых сессий. Задачи ведутся в context.md → СЛЕДУЮЩИЙ ШАГ

---

## Формат context.md

```markdown
# НАЗВАНИЕ ПРОЕКТА

## ЦЕЛЬ
[что это и зачем, 1-3 предложения]

## СТАТУС
🟢 Работает / 🟡 В процессе / 🔴 Заморожен — [краткое описание]

## ЧТО СДЕЛАНО
- [ГГГГ-ММ-ДД] действие → результат

## СЛЕДУЮЩИЙ ШАГ
[конкретное действие, не «подумать», а «сделать X»]

## ГРАБЛИ
[что не работает и как правильно — только неочевидное]

## ТЕХНИЧЕСКИЕ ДЕТАЛИ
[пути, порты, команды запуска, зависимости]
```

## Правила ведения файлов

- `brain/grabli.md` — только дополнять снизу, **никогда не удалять**
- `brain/projects.md` — обновлять после каждой сессии
- `brain/workstations.md` — обновлять при настройке MCP, локальных инструментов, путей, ключей и различий между рабочим/домашним ПК
- `context.md` проекта — перезаписывать статус, дополнять «что сделано»
- Все пути в файлах — **относительные**
- Именование папок: `kebab-case` (не `CamelCase`, не `snake_case`)

## Git-дисциплина

- Старт сессии: `git pull`
- После каждой значимой задачи: коммит
- Конец сессии: `git push`
- Команда «Переходим в новый чат»: обновить контекст, memory graph, саммари сессии, сделать commit + push и выдать готовое сообщение для нового чата
- Сообщение коммита по-русски — норма
- Бинарные файлы (obj/, bin/, __pycache__/) — всегда в .gitignore
