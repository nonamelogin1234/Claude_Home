# ПРАВИЛА РАБОТЫ

## Файлы проектов

Каждая папка проекта содержит:
- `context.md` — цель, статус, что сделано, следующий шаг, грабли, технические детали
- Код проекта

Формат записи в «что сделано»:
```
- [ГГГГ-ММ-ДД] ДЕЙСТВИЕ → РЕЗУЛЬТАТ
```

## Правила ведения файлов
- `brain/grabli.md` — только дополнять снизу, никогда не удалять
- `brain/projects.md` — обновлять статус после каждой сессии
- `context.md` проекта — перезаписывать статус, дополнять «что сделано»
- Все пути в файлах — относительные

## Git-дисциплина
- Старт сессии: `git pull`
- После каждой значимой задачи: коммит с понятным сообщением
- Конец сессии: `git push`
- Сообщение коммита по-русски — норма

## Шаблон нового проекта (context.md)

```markdown
# НАЗВАНИЕ ПРОЕКТА

## ЦЕЛЬ
[что это и зачем]

## СТАТУС
[эмодзи + текст]

## ЧТО СДЕЛАНО
- [дата] действие → результат

## СЛЕДУЮЩИЙ ШАГ
[конкретное действие]

## ГРАБЛИ
[что не работает и как правильно]

## ТЕХНИЧЕСКИЕ ДЕТАЛИ
[пути, порты, команды, зависимости]
```

## Для Claude Chat (claude.ai)
- Читает контекст только по команде «читай контекст» / «загрузи контекст» / «инициализируйся»
- Читает из репо Claude_Home через GitHub MCP
- Саммари важных сессий сохраняет в `sessions/ГГГГ-ММ-ДД_тема.md`
- Папки проектов целиком не читает если не нужно (они большие)
