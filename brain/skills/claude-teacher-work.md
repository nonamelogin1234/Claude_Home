## Подключение к серверам

### Shell API — запуск команд на VPS (для Клода)

```powershell
powershell -ExecutionPolicy Bypass -File C:\Users\torganov-a\srv.ps1 -cmd "КОМАНДА"
```

Альтернатива — напрямую через API:
```
POST https://mcp.myserver-ai.ru:7723
X-Secret: shell-api-secret-2026
{"cmd": "команда"}
```

> ⚠️ С рабочего ПК доступен только VPS через shell API. Домашний сервер через shell API недоступен.

### Shell-команды на домашнем сервере

С рабочего ПК доступ к домашнему серверу только через SSH (через VPS).
Клод может запустить команду так:

```powershell
powershell -ExecutionPolicy Bypass -File C:\Users\torganov-a\srv.ps1 -cmd "ssh -o StrictHostKeyChecking=no sergei@10.8.0.27 'КОМАНДА'"
```

> ⚠️ SSH-ключ VPS→домашний сервер может быть не настроен. Если не работает — уведоми Сергея.

### SSH для Сергея — терминал на серверах

**VPS:**
```bash
ssh sergei@147.45.238.120
```
SSH-ключ с рабочего ПК добавлен в authorized_keys на VPS (апрель 2026). Работает без пароля.

**Домашний сервер (вариант 1 — ProxyJump, одна команда):**
```bash
ssh -J sergei@147.45.238.120 sergei@10.8.0.27
```

**Домашний сервер (вариант 2 — два шага):**
```bash
ssh sergei@147.45.238.120   # шаг 1: войти на VPS
ssh sergei@10.8.0.27        # шаг 2: с VPS на домашний сервер
```

> ℹ️ nginx на VPS не имеет модуля stream — TCP-проксирование SSH не работает.
> ℹ️ Порт 7724 — это shell API домашнего сервера, НЕ SSH.

## Железные правила (не меняются никогда)

1. **Не логировать сессию без явного разрешения Сергея.** Спросить в конце.
2. **Сергей вводит команды сам.** Ты показываешь команду — он вводит. Не писать «скопируй и выполни».
3. **Не давать готовый ответ когда ученик близок к выводу.** Задать вопрос вместо ответа.
