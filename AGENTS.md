# Codex — ТОЧКА ВХОДА

> Читается автоматически при старте каждой сессии Codex.

## Контекст (загружается автоматически через @import)

См. @brain/infra.md — серверы, порты, подключения.
См. @brain/grabli.md — что не делать.
См. @brain/projects.md — статусы всех проектов.
См. @brain/rules.md — структура репо и правила.
См. @brain/coding.md — правила написания кода.
См. @brain/workstations.md — MCP, локальные инструменты и различия рабочих мест.

## Структура репозитория

```
projects/   ← код-проекты (kinoclaude, docai, rpg-tracker, revit-mcp, grok-news)
infra/      ← инфраструктура (homepage, vpn-hide, dashboard, navidrome, homeserver, ...)
archive/    ← замороженные проекты
brain/      ← системные знания (всегда читать)
sessions/   ← саммари сессий Codex.ai
```

## Старт сессии
1. Выполни `/project:start` или вручную: `git pull`
2. Если работаем над проектом — прочитай его `context.md`
3. Напиши одной строкой: «Прочитано: brain/ + [проект если был]»

## Конец сессии
По команде «пуш» или «конец сессии»:
1. Обнови нужные файлы в `brain/` и папке проекта
2. `git add -A && git commit -m "описание" && git push`

## Переход в новый чат
По точной команде «Переходим в новый чат» выполнить ритуал без длинных уточнений:
1. Обновить `context.md` активного проекта, `brain/projects.md`, при необходимости `brain/grabli.md`, `brain/workstations.md`, `brain/memory-graph.jsonl`.
2. Создать/обновить саммари в `sessions/`.
3. Сделать промежуточный commit и `git push`, не трогая чужие незакоммиченные изменения без причины.
4. В конце выдать готовое сообщение для нового чата, чтобы пользователь мог его вставить и продолжить с того же места.

## Правила
- Пути всегда относительные
- Грабли только дополнять, никогда не удалять
- После каждой задачи — коммит
- Язык общения: русский
- Перед новым проектом/фичей — архитектурное обсуждение (см. brain/coding.md)
- Если на одном компьютере настроили MCP/инструмент, фиксировать это в `brain/workstations.md`.
- Не смешивать личности: Codex — это текущий рабочий инженерный напарник в этом чате; Коготь — отдельный OpenClaw-агент в Telegram на homeserver.

## Compact Instructions
При сжатии контекста сохранять: текущую задачу, статус проекта, последние грабли.
