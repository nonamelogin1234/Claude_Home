# CLAUDE CODE — ТОЧКА ВХОДА

> Этот файл читается автоматически при старте каждой сессии Claude Code.

## Старт сессии

1. Сделай `git pull` чтобы получить актуальные файлы
2. Прочитай файлы из `brain/`:
```
cat brain/infra.md
cat brain/grabli.md
cat brain/projects.md
cat brain/rules.md
```
3. Если работаем над конкретным проектом — прочитай его папку (например `docai/context.md`)
4. Напиши одной строкой что прочитано: «Прочитано: brain/ + [проект если был]»

## Конец сессии (команда «пуш» или «конец сессии»)

1. Обнови нужные файлы в `brain/` и папке проекта
2. Сделай `git add . && git commit -m "[описание]" && git push`
3. Сообщи что обновлено

## Структура репо

```
Claude_Home/
├── CLAUDE.md          ← этот файл (точка входа)
├── brain/             ← общая память (читать всегда)
│   ├── infra.md       ← серверы, порты, подключения
│   ├── projects.md    ← статусы всех проектов
│   ├── grabli.md      ← что не делать
│   └── rules.md       ← правила работы с файлами
├── docai/             ← проект DocAI
└── kinoclaude/        ← проект КиноКлод
```

## Правила

- Пути всегда относительные (без C:\\Users\\... и /root/...)
- После каждой значимой задачи — коммит
- Грабли дополнять, никогда не удалять
